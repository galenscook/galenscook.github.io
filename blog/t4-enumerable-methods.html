Exploring the World of Enumerables

The .map Method

07|12|15

DBC took the first week of July off, during which period much of my acquired skill rusted and leaked out of my brain. This combined with the more advanced Ruby material made diving in to Week 4 somewhat of a challenge. After being overwhelmed by the first challenge, I put some major time into research and the code started flowing again. I'd briefly explored the Enumerable Ruby docs during Week 3, but this week I spent a lot of time scanning through those methods and figuring out how to make them work in my solutions. One that I used a few times and can definitely see major potential for is the map method, which is the subject of today's post.

map

Let's start by considering the following array: 

array[1, 2, 3]

In general, Enumerable methods all act like each. That is to say, they do something to each element of a collection, one at a time. I've said that map seems so useful because it's purpose is simply (and broadly) to change the elements based on some set of rules. So, back to our array, if we realized that they should all be negative numbers rather than positive, we could change them all at once like this:

 array.map {|x| -x} #=> [-1, -2, -3]

Now, something important to note, is that map is a non-destructive method. That is, it does NOT alter the original array. Instead, it creates a NEW, transformed array. Because of this, I've found it helpful to define a new variable, that way both the original and the transformed arrays are easily accessible.

 array = [1, 2, 3] 

  new_array = array.map{|x| -x} 

            

  p array #=> [1, 2, 3]

  p new_array #=> [-1, -2, -3]

A couple things to note about map:

  • If you want to transform the original array rather than creating a new one, you can use the map! variation.
  • Once you get comfortable with the basics of map, try experimenting with the code block to target specific elements within the array. If you want to embed an if method, try using the streamlined ? : notation!

After this week, I still feel like I've only scratched the surface of Enumerables' capabilities--but hey, baby steps!. As a closing piece of advice, Ruby docs are your friend! Don't be intimidated by them. Try to find a method that looks promising, and then do some extra research to figure out how to implement it. Little by little, we'll all get better!

Happy coding!
