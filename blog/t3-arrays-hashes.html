Arrays and Hashes

Lists on lists

06|28|15

This week we dove into the magical world of Ruby. Ruby is one of many programming languages, and has been labeled one of the easiest to learn, largely do to it's simplicity. Well, I'll say "easiest to learn" and "simplicity", because in the process of completing my challenges this week those weren't always my first choices of words. But, I did like that once I finally found a solution to the challenge I could go back and figure out exactly what each piece of code was doing. A lot of the commands are exactly what you'd expect them to be, which is nice.

Our first couple challenges used really basic input. One word or one number, then do something to it. Quickly identifying the limitations of this kind of program, however, we learned about arrays and hashes. At their cores, arrays and hashes are pretty much the same thing. They are used to organize lists of data. Each item in the list is assigned a "slot", which you can use to "call" the item later in the program. These are called key-value pairs. The key refers to the slot

The only difference lies in what the key is. Information in an array is slotted by an index number. The first item in the array is in slot zero, and it counts up from there. (Yes, slot zero, not slot one.) So, if you create an array such as: 

array["bob", "shoe", "horse"]
You could then call slot 1 like this: 
array[1]
Which means you'd be calling "shoe". (The numbering takes a little while to get used to).

As you can see, arrays are based on an order. In the array we've created above, the order seems pretty arbitrary. Why is bob in slot zero? What does that tell us about the data? For lists that aren't necessarily in an order, a hash often makes more sense. Rather than using numbers as keys, you (as the programmer) get to assign each value a key that you then use to call the value. Take a look at the table to the right below. Doesn't it make much more sense to use this to call the "horse" value? 

hash{animal}

Arrays
Key Value
0 bob
1 shoe
2 horse
Hashes
Key Value
name bob
clothing shoe
animal horse

So that's the gist of arrays and hashes. Both used to store lists of information, with the biggest difference being how that information is indexed. I should mention that the methods you can use to manipulate information in arrays and hashes are largely the same. I've used mostly arrays in my limited experience, but it seems to me that hashes are more robust and will (once I get that far) provide more functionality. We shall see!

Happy coding!
