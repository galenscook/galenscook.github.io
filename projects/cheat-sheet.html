JavaScript Objects Cheat Sheet

←Back to project list

Objects in JavaScript, just as in many other programming languages, can be compared to objects in real life. The concept of objects in JavaScript can be understood with real life, tangible objects.

In JavaScript, an object is a standalone entity, with properties and type. Compare it with a cup, for example. A cup is an object, with properties. A cup has a color, a design, weight, a material it is made of, etc. The same way, JavaScript objects can have properties, which define their characteristics.

Here is an example of an empty object. 

var pizza = {};

Let's take a look at how to add things to our pizza object. JavaScript offers two ways to do this: object literal notation, and dot notation. Below you'll find a guide for both.

Object Literal Notation

Note that with object literal notation, the properties and values are written as strings, as the syntax requires quotation marks.

Adding Properties and Values

Description of all the toppings we want to add, and the sauce. 

pizza["topping1"] = "cheese";
pizza["topping2"] = "pepperoni";
pizza["sauce"] = "light sauce";
pizza["crust"] = "hand-tossed";

pizza = { 
   topping1: "cheese",
   topping2: "pepperoni",
   sauce: "light sauce",
   crust: "hand-tossed"}

Object Dot Notation

Note that with object dot notation, the dot indicates that what follows is variable name. This changes the way dot notation is used in other levels of objects.

Adding Properties and Values

Description of all the toppings we want to add, and the sauce. 

pizza.topping1 = "cheese";
pizza.topping2 = "pepperoni";
pizza.sauce = "light sauce";
pizza.crust = "hand-tossed";

pizza = {
   topping1: "cheese",
   topping2: "pepperoni",
   sauce: "light sauce",
   crust: "hand-tossed"}

Removing Properties and Values

If you decide you don't want pepperoni on your pizza, you can always take it off. You can keep the topping2 property and leave it with an undefined value, or simply remove the property and value pair altogether. Here's how you would keep the property but remove the value, and what this change does to our object when we print it out. 

pizza["topping2"] = undefined;

// now that we've removed the pepperoni
// here's how our object looks

pizza = {
   topping1 = "cheese",
   topping2 = undefined,
   sauce = "light sauce",
   crust = "hand-tossed" }

Our topping2 property has an undefined value, which is the same as saying our pizza could have a second topping, it just doesn't happen to. An undefined value is still a value after all. It's just not very meaningful. If you wanted to remove the option for a second topping altogether, you would need to remove the value and the property. Here's how you would do it, and what this change would do to our pizza. 

delete pizza["topping2"];

// delete object[property] removes a
// property and its value. here's what
// our pizza object looks like now

pizza = {
   topping1 = "cheese",
   sauce = "light sauce",
   crust = "hand-tossed" }

Now our pizza object only has three properties and three values. It's a pie fit for an pizza-loving vegetarian.

Changing Property Values

Object properties must be unique. Because of this, changing a property's value looks identical to adding a property. 

// This is our current pizza.
pizza = {
   topping1: "cheese",
   sauce: "light sauce",
   crust: "hand-tossed"}

//If instead we want thin crust, we do this.
pizza["crust"] = "thin";

//This is our altered object.
pizza = {
   topping1: "cheese",
   sauce: "light sauce",
   crust: "thin"}

Nesting Objects within Objects

Sometimes, a property has more than one value. When this happens, you can define the property as it's own object, thereby creating a nested object. If we wanted multiple toppings on our pizza, rather than naming a bunch of separate properties, we could nest them! 

//If we wanted to add toppings, we could do so by making the toppings property an object itself.

pizza["toppings"] = {};
pizza["toppings"]["topping1"] = "cheese";
pizza["toppings"]["topping2"] = "pepperoni";
pizza["toppings"]["topping3"] = "peppers";

//This is our new nested object.
pizza = {
   toppings: {
      topping1: "cheese",
      topping2: "pepperoni",
      topping3: "peppers"},
   sauce: "light sauce",
   crust: "thin"
}

//To access just the second topping, we would type this.
pizza["toppings"]["topping2"] = "pepperoni"
